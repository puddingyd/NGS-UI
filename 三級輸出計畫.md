## 1. 預計Pipeline 輸出（UI 輸入規格）

三級分析最終輸出為 **JSON + TSV**，供 UI 讀取：

```
/home/pipeline/tertiary_output/
├── _index.json                     ← 樣本清單（UI 頂部 combobox 用）
└── {SAMPLE_ID}/
    ├── snv_indel.annotated.tsv     ← SNV/Indel 主表（含所有 evidence）
    ├── cnv.annotated.tsv           ← CNV 主表
    ├── sv.annotated.tsv            ← SV 主表
    ├── str.classified.tsv          ← STR 判定結果
    ├── mito.annotated.tsv          ← mtDNA 主表
    ├── roh_summary.json            ← ROH 摘要（consanguinity, UPD flags）
    ├── roh_regions.tsv             ← ROH 區段清單（HomRegions.tsv 增強版）
    ├── secondary_findings.tsv      ← ACMG SF v3.2 結果
    ├── pgx.tsv                     ← PGx haplotype + drug recommendation
    ├── lirical_results.tsv         ← LIRICAL gene ranking
    ├── exomiser_results.tsv        ← Exomiser gene ranking
    ├── qc_summary.json             ← 來自二級分析的 QC metrics 摘要
    └── sample_metadata.json        ← sample_id, seq_type, hpo, run_date 等
```

### _index.json 樣本清單格式

由 pipeline 跑完每個 sample 時 append 一筆，或由 UI 後端定期掃 `tertiary_output/*/sample_metadata.json` 重建。前端搜尋以 `lis_id` / `name` / `mrn` 三欄聯合 fuzzy match。

```json
[
  {
    "sample_id": "26WE0064",
    "lis_id": "26WE0064",
    "name": "孫亮葇",
    "mrn": "23293706",
    "test_type": "WES",
    "category": "MCA",
    "run_date": "2026-04-18",
    "tags": [],
    "yield": 1,
    "has_completed": true,
    "tertiary_dir": "26WE0064"
  }
]
```

欄位說明：
- `sample_id` — pipeline 內部 ID，等於子目錄名稱
- `lis_id` — 檢驗單號（給諮詢師查找用，常與 sample_id 相同）
- `mrn` — 病歷號（給 EMR 連結用）
- `test_type` — `WES` / `WGS`
- `category` — 病歷分類（如 `MCA` / `epilepsy` / `PND`），UI 下拉選單值來自 `_options.json` 的 `category_options`
- `tags` — 自由標記（多選），同上由 `_options.json` 提供建議清單
- `has_completed` — pipeline 是否完成；UI 顯示 spinner 或灰階
- `tertiary_dir` — 相對於 `tertiary_output/` 的子目錄名稱

### _options.json（選用）

```json
{
  "category_options": ["MCA", "epilepsy", "intellectual_disability", "PND", ...],
  "tag_suggestions":  ["pending_consult", "trio_needed", "follow_up", ...]
}
```

### snv_indel.annotated.tsv 欄位規格（★ v2.4 更新）

```
CHROM, POS, REF, ALT,
GENE, TRANSCRIPT, TRANSCRIPT_TYPE (MANE_SELECT/MANE_PLUS_CLINICAL/CANONICAL),
HGVS_C, HGVS_P, CONSEQUENCE,
MANE_ALL (JSON 字串，含所有 MANE transcript 的後果，供 GUI 展開),
CALLERS (DV+HC / DV / HC),
ZYGOSITY, GT_DV, GT_HC,
AD (allele depth, e.g. "21,18"), VAF (alt allele fraction, 0–1),
CLINVAR_SIG, CLINVAR_STARS, CLINVAR_DN, CLINVAR_CONF,

TWB_AF (預留空白),
PKNN_LLR (missense PP3/BP4 主要依據),
REVEL, BAYESDEL, ALPHAMISSENSE, METARNN (保留供參考),

PHASE_GROUP, PHASE_RESULT (trans/cis/unphased),
IN_ROH, IN_PANEL, IN_BLACKLIST,
OMIM_LINK, GNOMAD_LINK, CLINVAR_LINK,
REPORT_CLASS (空白/1/2/3, UI 填入)
```

## 2. UI 架構與技術選型 -> 這部分你可以照claud習慣的方法更改，下面只是我的習慣，但如果用到npm，注意版本要用最新的，之前有個版本很容易攻破

### 2.1 技術

| 層級 | 技術 | 理由 |
|------|------|------|
| 前端 | React + TypeScript | 元件化，複雜表格操作 |
| 表格 | AG Grid Community（MIT）| Excel-like，排序/篩選/顏色/選取 |
| 後端 | FastAPI（Python）| 與 bioinformatics 工具鏈整合佳 |
| 資料庫 | PostgreSQL | Panel、blacklist、報告草稿、使用者設定 |
| 部署 | Docker Compose | 院內 server，瀏覽器存取 |

### 2.2 資料載入流程

```
三級分析輸出 TSV/JSON
        ↓
FastAPI 後端（/api/load_sample）
  ├── 讀取 snv_indel.annotated.tsv → 轉為 JSON
  ├── 讀取 cnv/sv/str/mito TSV
  ├── 讀取 roh_summary.json
  ├── 讀取 qc_summary.json
  └── 寫入 PostgreSQL session table
        ↓
React 前端（AG Grid）
  ├── GET /api/variants/{sample_id}/snv
  ├── GET /api/variants/{sample_id}/cnv
  └── ...
```

#### Page 2：SNV/Indel 篩選（★ v2.4 更新：transcript 展開）

**排序優先順序：**

| 大類 | 底色 | 條件 |
|------|------|------|
| 1A | 🔴 深紅 | ClinVar P/LP ≥ 1★ |
| 1B | 🔴 紅 | Frameshift / nonsense（LOFTEE HC）|
| 1C | 🔴 中紅 | ACMG points ≥ 4 |
| 2 | 🩷 淺紅 | ClinVar P/LP 0★ 或 Conflicting（含 P）|
| 3 | 🩷 淡紅 | 其餘（ACMG points < 4）|

**AG Grid 功能設定：**
- 多欄排序（先 ClinVar tier，後 ACMG points）
- Row 顏色依大類自動設定
- **Transcript 展開**：點擊 row 左側展開箭頭 → 顯示 MANE_ALL 中所有 MANE transcript 的後果（master-detail 模式）
- TRANSCRIPT_TYPE 欄位顯示 `MANE_SELECT` / `MANE_PLUS_CLINICAL` / `CANONICAL` badge
- 點擊 Gene → 自動 filter 同基因所有 variants（輔助 recessive 判讀）
- 每個 variant 最右側有下拉選單：`空白（default）/ 報告類別一 / 報告類別二 / 報告類別三`
- 選擇後以顏色 badge 顯示於該 row
- 右鍵 context menu：
  - 開啟 ClinVar 連結
  - 開啟 OMIM 連結
  - 開啟 gnomAD 連結
  - 標記需人工審閱

#### Page 3：CNV/SV

- 同 Page 2 架構
- 額外欄位：Type（DEL/DUP/INV/BND）、Size（Mb/kb）、Genes_overlapped、DGV_freq、AnnotSV_rank
- CNVkit scatter plot 和 diagram 可在側欄開啟（PDF embed）

#### Page 4：STR

- 只顯示 `PATHOGENIC` 或 `INTERMEDIATE` 的 loci
- 欄位：Gene, Disease, Repeat_unit, Allele1, Allele2, Normal_max, Pathogenic_min, Status
- 特殊 loci（FMR1/RFC1）標記「需確認 motif」

#### Page 5：ROH Viewer

- 上方：Ideogram（每條染色體的 ROH 區段視覺化，顏色深淺代表大小）
- 中間：Consanguinity 評估 + UPD 候選染色體
- 下方：ROH 清單表格（Chr, Start, End, Size, Genes，可點擊查詢 gene 是否在 panel 內）

#### Page 6：Secondary Findings

- 固定 ACMG SF v3.2 gene list
- 只顯示 ClinVar Pathogenic（≥ 1★）
- 欄位：Gene, Variant, HGVS, ClinVar_ID, Disease, Actionability, OMIM_link
- 有獨立的「加入報告」選項（與主要報告分開）

#### Page 7：Pharmacogenomics

- 固定 CPIC gene panel
- 欄位：Gene, Diplotype, Phenotype, Drug, Recommendation, Evidence_level
- 提供 CPIC guideline 超連結

#### Page 8：報告產生

```
┌─ 選取的 Variants ──────────────────────────────────┐
│  類別一 variants：[來自 Page 2-4 選擇的類別一]     │

│  類別三 variants：[來自 Page 2-4 選擇的類別三]     │
│  Secondary Findings：[來自 Page 6]                 │
```
